Events to listen

verifyUser event
This event ("verifyUser") has to be listened on the user's end to verify user authentication.
userId event
This event ("userId") has to be listened on the user's end to identify user with same userId will receive friend request received notification.
The output sent from backend can be shown as toast meessage for reflecting real-time-notification.
This event's name is actually dynamic and refers to the userId of the logged in user.
'fRAccept'+userId
This event's userId is actually dynamic and refers to the userId of the logged in user.
This event ("'fRAccept'+userId") has to be listened to identify an individual userId that will receive friend request accepted notifcation.
'fRAccept' is added to dyanamic userId to avoid sending notification to event "userId" which may create conflict
and result in notification sending to wrong user.
'create' + userId
This event's userId is actually dynamic and refers to the userId of the logged in user.
This event ("'create' + userId") has to be listened to identify an individual userId that will receive multi todo creation notifcations in case of multi todos creation.
and todo changes notification in case if there is any change in multi todos
'create' is added to dyanamic userId to avoid sending notification to event "userId" which may create conflict
and result in notification sending to wrong user.

Events to emit

set-user event
This event ("set-user") has to be emitted when a user login to the app.
User can only be set as online and eligible to recieve app notifications only after verification of authentication token(authToken). Which you have pass here. The following data has to be emitted:
authentication token
friend-info
This event ("friend-info") has to be emitted, when user sends a friend request
The following data has to be emitted
                            {
                                userName:  'senderName',
                                senderId:  'sender's userId',
                                receiverId:'receiver's userId',
                                sentOn: 'requestSentOn',
                                status: 'requestStatus'
                            }
accept-request
This event ("accept-request") has to be emitted, when user has accepted the friend request
The following data has to be emitted
                            {
                                userName: 'userName of receiver',
                                senderId: 'sender's user id',
                            }
accept-request
This event ("multi-todo-create") has to be emitted, when user has either created a multi todo or edited a multi todo
The following data has to be emitted
                            {
                                message: 'remarks', //example remarks=`${this.userName} changed the title from ${title} to ${this.titleEdited}`
                                senderId: 'current user's userId'
                            }
logout
This event ("multi-todo-create") has to be emitted, when user has hit the logout api call
